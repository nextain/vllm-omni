## English

This directory contains scripts and benchmarks for serving **MiniCPM-o 4.5** via vllm-omni's
OpenAI-compatible API. MiniCPM-o 4.5 is a 3-stage omni model:
**Thinker** (multimodal LLM) → **Talker** (TTS codec generator) → **Code2Wav** (CosyVoice2 vocoder).

### Quick Start

```bash
# 1. Start the server (2× RTX 3090)
NCCL_P2P_DISABLE=1 vllm serve openbmb/MiniCPM-o-4_5 --omni \
  --stage-configs-path ../../vllm_omni/model_executor/stage_configs/minicpmo_async_chunk.yaml \
  --trust-remote-code --host 0.0.0.0 --port 8000

# 2. Run the conversation benchmark
cd examples/online_serving/minicpm_o/
python conversation_benchmark.py --omni

# 3. Run VoiceBench evaluation
python voicebench_runner.py
```

### Scripts

| Script | Purpose |
|--------|---------|
| `conversation_benchmark.py` | Multi-turn conversation benchmark across 6 English scenarios |
| `language_test.py` | EN / ZH / KO comparison with CER + semantic similarity |
| `voicebench_runner.py` | VoiceBench (Knowledge / Instruction / Robustness / Safety) |
| `e2e_conversation_test.py` | Core speaker / monitor framework (imported by above scripts) |
| `metrics/cjk_metrics.py` | CER, semantic similarity (sentence-transformers) |
| `metrics/conversation_quality.py` | Relevance, coherence, knowledge retention |

### Known Limitations

- **Stop token 6561** rarely generated by Talker → audio duration fixed at ~20s
  (tracked separately; workaround: `_trim_silence()` post-processing)
- **Korean TTS**: CosyVoice2 not trained on Korean → garbled audio
- **async_chunk TTFP**: First-audio latency ~0.07s. Note: benchmark client's
  `modality=="audio"` detection may miss streaming chunks — see `e2e_conversation_test.py`.
